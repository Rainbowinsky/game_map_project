# P2-6 缩略导航与验收收口实施报告

P2-6 已交付独立的缩略导航投影、键盘可达的画布定位和第二阶段的性能观测场景。缩略图只从已加载的领域对象与当前相机状态生成低分辨率 Canvas 图形；它不调用 PNG 导出、不读取 GPU 像素，也不成为地图数据的真源。

## 交付内容

- `Minimap` 绘制地图边界、区域、路径、地形笔迹、文字、marker 和图章的简化概览，并叠加当前可视矩形。
- 点击或拖动缩略图会在当前缩放级别平移相机；目标中心会被限制在地图边界内。缩略图隐藏后会停止绘制工作。
- 缩略图画布可聚焦，支持方向键移动、Shift + 方向键大步移动和 Home 回到地图中心；展开/收起控制带有明确的 ARIA 标签与状态。
- 新增 500/2,000 个混合几何对象的确定性场景，`pnpm --filter @fantasy-map/web test:benchmark` 同时运行现有图章投影和缩略导航坐标投影基准。基准记录可复现吞吐量，不设置不可靠的跨机器 FPS 门槛。
- Playwright 覆盖缩略图的 ARIA 描述、键盘定位、点击定位和显示/隐藏流程；单元测试覆盖坐标往返、可视矩形裁剪和边界夹紧。

## 边界与验收

- 缩略导航只处理当前已加载对象，继续遵守第二阶段不承诺 Chunk 流式加载、空间索引或 LOD 的边界。
- 相机定位不生成 command、operation 或自动保存批次；它仅改变浏览器视图状态。
- 验收运行：`pnpm lint`、`pnpm typecheck`、`pnpm test`、`pnpm build`、`pnpm test:e2e` 和 `pnpm --filter @fantasy-map/web test:benchmark`。API、Prisma 和 storage 未在 P2-6 变更，因此不新增迁移或集成测试场景。
