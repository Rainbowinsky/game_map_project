# 第二阶段：绘制能力准备索引

状态：P2-0、P2-1、P2-2 已完成；P2-3 至 P2-6 尚未开始功能实现。

第二阶段把第一阶段的图章编辑器扩展为可绘制、可标注、可使用自有素材的地图编辑器。实现必须保持第一阶段已经验证的约束：MySQL 是长期真源、编辑写入继续使用带 `revision` 与幂等 mutation ID 的 operation 批次、PixiJS 只负责投影渲染、客户端不提交存储路径。

## 范围

1. 地形笔刷与区域多边形。
2. 可编辑的河流、道路路径。
3. 文字与地点资料。
4. 自定义图片素材上传、分类和安全访问。
5. 可替换主题与缩略导航图。

本阶段不包含可见 Chunk 流式加载、空间索引、LOD、Worker、分块高清导出、后台任务、公开地图或多人协作；这些仍属于后续阶段。第二阶段可在当前“已加载地图对象”的边界内交付，但不能把不可编辑的位图烘焙结果当作源数据保存。

## 已确认的基线

- `MapLayer` 已有 `raster`、`vector-path`、`text`、`region`、`marker` 类型；不需要为本阶段重做图层树。
- Prisma 已有 `Asset`、`AssetCategory`、`Location` 表以及安全的 `StorageProvider`；上传模块需在这些边界上实现，而不是接收客户端文件路径。
- 当前 `MapObject` 仅支持 `stamp`，operation、映射与 Pixi 投影都需要在同一工作包内扩展为判别联合。
- 当前主题值仍位于 `map-artwork.ts`、网格和选择叠层中；主题注册表必须先落地，再迁移这些值，不能在工具代码中继续新增颜色常量。

## 开工顺序

先完成 P2-0 契约冻结与测试夹具，再依次实现主题基础、路径/区域、地形笔刷、文字/地点、素材上传，最后接入缩略图并完成端到端验收。每个工作包都必须在合入前通过 `lint`、`typecheck`、相关单测和构建；涉及 API/数据库时额外通过集成测试。
