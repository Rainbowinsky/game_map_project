# P2-3 地形笔刷实施报告

P2-3 已在 v2 `terrain-stroke` 契约上交付地形笔刷与整笔橡皮擦。浏览器按屏幕像素捕获普通指针及合并指针事件，再按世界距离重采样；持久化数据始终是可编辑的语义中心线、笔刷参数和随机种子，不保存画布位图或 Pixi 指令。

## 交付内容

- 工具栏新增地形笔刷（B）和地形橡皮擦（E），支持水域、森林、山地、沙漠和草地类型。
- 图层面板可创建 `raster` 地形图层。工具优先写入当前可编辑地形图层，没有可用图层时明确提示，不跨类型写入。
- 笔刷面板支持半径、不透明度、硬度与采样间距，可收起、重新展开，并可通过标题栏或收起后的按钮在画布范围内拖动避让。绘制消费浏览器的 coalesced pointer events，保留笔压，并以至少一个屏幕 CSS 像素的输入粒度转换为世界坐标。
- 普通绘制过程仅更新本地 Pixi 预览；pointer up 才通过 `DrawTerrainStrokeCommand` 提交一个 `object.create` operation，因此正常笔势只产生一个历史条目和一个自动保存批。超长笔势达到单对象字节上限时会立即把当前安全段落提交到画板和保存队列，并从共享端点继续下一段，避免整条笔势在松开时丢失。
- 重采样算法沿长指针移动插值补点，提交时保留精确末端坐标；单击也形成一个屏幕像素长度的有效笔迹。长笔势在提交前使用屏幕精度内的迭代 RDP 自适应简化，保留转折、端点和显著笔压变化并压入 JSON 字节预算；极端输入仍由统一 schema 安全拒绝并显示可读提示。
- 橡皮擦显示半透明扫掠预览，使用线段最短距离并同时考虑橡皮擦与原笔刷半径。第一版按设计删除相交 stroke 整体；一次拖动命中的多条笔迹合并为一个事务历史项和一个 operation 批，可整体撤销。
- `ObjectProjection`、选择命中、可见区域裁剪和 PNG 导出已接入地形笔迹。实时编辑器与导出共用绘制函数和解析后的主题 token；硬度以软边外层和实心内层投影。

## 数据与兼容边界

- 未新增数据库迁移或 API 路由；保存继续使用 `POST /maps/:mapId/operations` 和既有严格 `terrain-stroke` schema。
- `brush.spacing` 与点列使用世界单位；像素只用于输入精度和当前缩放下的最小采样阈值，因此缩放、导出和刷新不会把笔迹退化为屏幕位图。超长笔势可对应多个首尾相接的语义 stroke，每个对象独立满足持久化预算。
- 橡皮擦不切割或重组中心线；碎片重组留给后续版本。锁定、隐藏或位于不可编辑图层的笔迹不会被擦除。
- 当前仍遵循全量已加载对象边界；空间索引、Worker 简化和远程 Chunk 调度属于第三阶段。

## 验收证据

- 重采样单测覆盖固定世界间距、笔压插值、精确末端、点数预算失败，以及超长笔势自动压入持久化字节预算。
- 相交测试覆盖笔刷半径合并后的命中与未命中。
- 命令测试验证一笔只发布一个 operation 批、只增加一个历史项，并可撤销。
- Web 全量单测、lint、typecheck 与生产构建通过；P2-3 不涉及 API、Prisma 或 storage，因此无需新增集成测试迁移场景。

## 后续边界

P2-4 可在相同命令、patch bus、operation journal 和增量投影链路上实现文字与地点。若后续加入笔迹局部擦除，应先定义确定性的中心线切割与最短片段策略，并继续保证单手势单历史/保存批。
